# What is Claude Code?

Imagine having a **senior developer sitting right next to you** — someone who never gets tired, never judges your questions, and is always ready to help. That's Claude Code.

Claude Code is an **AI assistant that lives in your terminal**. Instead of chatting in a browser window, you talk to it right where you write code. It can see your files, understand your project, and help you build things.

## What Can It Do?

Claude Code is like having a super-powered teammate. It can:

- **Create files** — ask it to make a new page, component, or config file
- **Write code** — describe what you want and it writes it for you
- **Fix bugs** — show it an error and it figures out the fix
- **Explain code** — point it at confusing code and it breaks it down in plain English
- **Answer questions** — ask it anything about your project or programming in general
- **Refactor code** — clean up messy files, split them into smaller pieces, add comments
- **Write tests** — generate automated tests to make sure things keep working

## How Is It Different from ChatGPT?

You might have used ChatGPT or Claude in a web browser before. Claude Code is different because:

1. **It sees your project** — it can read all your files and understand how they fit together
2. **It makes changes directly** — instead of copying and pasting, it edits your files for you
3. **It lives in your terminal** — no switching between browser tabs and your code editor
4. **It remembers project context** — it knows your file structure, your dependencies, and your patterns

Think of it this way: chatting with AI in a browser is like texting a friend for advice. Using Claude Code is like having that friend **sit down at your computer** and help you hands-on.

With ChatGPT, you copy code into a chat box, get a response, then copy it back. With Claude Code, it reads your actual files, makes changes in place, and even runs commands for you.

## What Teams Build with Claude Code

Real teams use Claude Code for serious work every day:

- **Marketing teams** build landing pages and email templates without waiting for developers
- **Small agencies** create WordPress sites and custom plugins in a fraction of the time
- **Startups** prototype new features quickly — describe what you want and see it built in minutes
- **Solo founders** build entire web applications without hiring a development team

## A Day in the Life

Here's what a typical day looks like for someone using Claude Code:

**9:00 AM** — Open your project, start Claude Code, ask it to review overnight changes and explain what happened.

**10:00 AM** — A client wants a new contact form on their website. You describe the form to Claude Code and it builds the whole thing — HTML, styling, and the code that sends emails.

**11:30 AM** — Something broke on the staging site. You paste the error message into Claude Code and say "fix this." It finds the bug and patches it.

**2:00 PM** — Time to add a new feature. You describe what you want in plain English, and Claude Code writes the code, creates the files, and even updates the navigation.

**4:00 PM** — You ask Claude Code to clean up the code from today, add comments, and write some tests.

## What Claude Code Can't Do

It's important to know the limitations:

- **No internet access** — Claude Code can't browse websites, check live APIs, or download things for you
- **No production deployments** — it shouldn't push code to your live site directly (always review first!)
- **Not always right** — it can make mistakes, especially with very new libraries or unusual setups
- **No memory between sessions** — when you close it and reopen, it starts fresh (unless you use a CLAUDE.md file)
- **Can't run your whole server** — it helps you write code, but you still manage your own infrastructure

## Why This Matters

Claude Code is the tool that makes everything in this course possible. You don't need years of programming experience. You need to know how to describe what you want clearly — and that's exactly what we're teaching you.

Think of Claude Code as a force multiplier. Your ideas, combined with its technical skills, let you build things that would normally take a professional developer days or weeks.

## Common Mistakes to Avoid

- **Being too vague** — "Make it better" doesn't help. "Make the heading larger and change the button color to blue" does.
- **Not reviewing changes** — always look at what Claude proposes before accepting. You're the boss.
- **Expecting perfection** — Claude Code is amazing, but it's not magic. Sometimes you'll need to try a few approaches.
