# What is an API?

You've been using APIs every single day — you just didn't know it. Let's fix that.

## The Restaurant Analogy

Imagine you're sitting in a restaurant. You want food, but you can't just walk into the kitchen and start cooking. Instead, you tell the **waiter** what you'd like. The waiter walks to the kitchen, places your order, and brings back your food.

That waiter? That's an API.

- **You** (the app) — you know what you want
- **The waiter** (the API) — carries your request and brings back the result
- **The kitchen** (the server) — does the actual work

API stands for **Application Programming Interface**. That's a fancy way of saying: **a way for apps to talk to each other**.

You don't need to know how the kitchen works. You just need to know how to talk to the waiter.

## APIs Are Websites for Computers, Not Humans

Here is another way to think about it. When you visit a website, you see a pretty page with colors, images, and buttons. That page is designed for **your eyes**.

An API is like a version of that website designed for **other programs** to read. Instead of pretty pages, it sends back raw data — just the facts, no decoration.

When you go to `weather.com`, you see a colorful forecast with icons. When an app uses the weather **API**, it gets back something like `{ "temperature": 72, "condition": "sunny" }`. Same information, different format — one for humans, one for computers.

## You Already Use APIs Every Day

Here are some examples you've probably used today:

- **Weather apps** ask a weather API for the forecast, and the API sends back the data
- **Uber** uses the Google Maps API to show you the map and calculate routes
- **Online stores** use a payment API (like Stripe) to process your credit card
- **Social media apps** use APIs to load your feed, post updates, and send notifications

Every time one app talks to another app behind the scenes, an API is making it happen.

## Real-World APIs You Should Know About

As you get into automation, you will run into these APIs constantly:

- **Stripe** — processes payments. When you buy something online, the store's website talks to Stripe's API to charge your card.
- **Twilio** — sends text messages and makes phone calls. Apps use it to send you verification codes or appointment reminders.
- **Google Maps** — provides maps, directions, and location data. Ride-sharing apps, delivery services, and real estate sites all use it.
- **OpenAI / Anthropic** — powers AI features. Apps send text to the API and get intelligent responses back (like Claude!).
- **Slack** — sends messages to channels. Automation tools use Slack's API to post alerts and notifications.
- **Google Sheets** — reads and writes spreadsheet data. Many automations use Sheets as a simple database.

These are just a few. There are **thousands** of public APIs out there, and more are created every day.

## REST vs. GraphQL: Two Styles of API

You might hear people mention **REST** and **GraphQL**. These are two different styles of building APIs.

- **REST** is the most common. You visit different URLs for different data — like `/posts` for blog posts and `/users` for users. It is straightforward and easy to understand.
- **GraphQL** lets you ask for exactly the data you want in a single request — like ordering a custom meal instead of picking from a set menu.

For now, you only need to know that REST exists — it is what most automation tools use. We will work with REST APIs throughout this course.

## Why Should You Care?

Because APIs are the foundation of automation. When you want AI to do something useful — like create a blog post, send an email, or update a spreadsheet — it uses APIs to make it happen.

Without APIs, every app would be an island. With APIs, apps can work together like a team.

## Try It Yourself

Want to see an API in action right now? Open your web browser and visit this URL:

```
https://api.github.com
```

You will see a page full of raw data — no pretty design, just text. That is what an API response looks like! It is GitHub's API telling your browser, "Here are all the things you can ask me about."

Now try this one:

```
https://api.github.com/users/octocat
```

You will see information about the GitHub user "octocat" — their name, profile picture URL, number of repos, and more. You just made your first API request!

> **Tip:** If the text looks messy in your browser, install a browser extension called "JSON Formatter" — it makes API responses much easier to read.

## Common Mistakes

- **Thinking APIs are only for programmers.** Not true! Automation tools like n8n, Make, and Zapier let you use APIs without writing code.
- **Confusing APIs with websites.** A website shows you pretty pages. An API gives you raw data. Same server, different format.
- **Being intimidated by the name.** "Application Programming Interface" sounds scary, but it is just a messenger between apps.

## Key Takeaway

An API is just a messenger between apps. You send a request, and the API brings back a response. That's it. No magic, no complicated science — just a waiter taking your order.
