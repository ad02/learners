# The WordPress REST API

Here's something cool: your WordPress site already has an API built right in. You don't need to install anything. It's just there, ready to go.

## Your Site Already Speaks API

Every WordPress site has what's called a **REST API**. This means other apps — and even AI — can talk to your WordPress site using the same request pattern you learned earlier (GET, POST, URLs, and so on).

You can:

- **Read posts** from your site
- **Create new pages** automatically
- **Update content** without logging into the dashboard
- **Manage users** programmatically

All without ever opening a browser.

## The URL Pattern

The WordPress REST API lives at a specific address on your site. The pattern looks like this:

```
yoursite.com/wp-json/wp/v2/posts
```

Let's break that down:

- **yoursite.com** — your website's address
- **/wp-json** — the starting point for the API
- **/wp/v2** — version 2 of the WordPress API
- **/posts** — what you want (posts, pages, users, etc.)

Want pages instead of posts? Just change the end:

```
yoursite.com/wp-json/wp/v2/pages
```

Want to see users?

```
yoursite.com/wp-json/wp/v2/users
```

It's that simple.

## Reading vs. Writing

**Reading** (GET requests) is open by default. Anyone can read your published posts through the API — just like anyone can read them on your website.

**Writing** (POST requests) requires authentication. You need an API key or login credentials to create or update content. This makes sense — you wouldn't want random people creating posts on your site!

## Try It Out

Use the explorer below to see how the WordPress REST API works. Try reading posts, then creating one.

        statusText: "Created",
        body: { id: 3, title: "New Post", status: "draft", message: "Post created successfully!" },

      }
    }
  ]}
/>

## The Automation Connection

This is where it all comes together. Remember what you've learned so far:

1. **APIs** let apps talk to each other
2. **Webhooks** trigger actions when events happen
3. **WordPress has a built-in API** for managing content

Now imagine this automation workflow:

1. **AI writes a blog post** (using a prompt you designed)
2. **The API creates the post** in WordPress automatically
3. **A webhook notifies your team** in Slack that new content is live

No manual work. No copying and pasting. No logging into WordPress. The whole thing runs on its own.

This is the power of connecting APIs, webhooks, and AI together. And in the next module, you'll learn how to build these kinds of workflows step by step.

## Key Takeaway

Your WordPress site already has a powerful API built in. You can read and write content programmatically, which means other tools and AI can interact with your site automatically. This is the bridge between "knowing about APIs" and "actually automating your work."
