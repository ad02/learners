# Your Final Project

Congratulations — you made it to the capstone!

Over the last eight modules, you went from zero to knowing how to use VS Code, Git, AI prompting, Claude Code, web basics, WordPress, APIs, and automation workflows. That is a LOT.

Now it is time to put it all together. You are going to build a **complete automation** from start to finish — a real workflow that connects multiple tools into one seamless process.

## How This Works

You will pick **one of three project templates** below. Each template is a guided, step-by-step walkthrough. You will not be left on your own — every step tells you exactly what to do and why.

Think of it like following a recipe. You have already learned what each ingredient does. Now you are combining them into a full dish.

## This Capstone is 100% Real

You'll build a working automation on real tools — not simulators. Everything runs on **free tiers**: Make.com, Groq AI, Google Sheets, built-in email. Total cost: $0.

When you're done, you'll have a live, 24/7 automation that processes real data and delivers real output.

## Choose Your Template

### Template A: Content Pipeline ⭐ (Recommended)

**What it does:** Receives submissions via webhook → Groq AI summarizes → logs to a Google Sheets draft queue → emails you a notification.

**Skills used:** Webhooks (Module 7), Groq API (Module 7), AI prompting (Module 3), Google Sheets integration (Module 8), email notifications.

**Best for:** Everyone. This is the **primary capstone** with the full step-by-step walkthrough. Start here.

---

### Template B: Daily Digest Bot (Variation)

**What it does:** Daily scheduled trigger → fetches data from a public API → Groq summarizes → logs to Google Sheets → emails.

**Skills used:** Scheduled triggers (Module 8), APIs (Module 7), Groq (Module 7), Google Sheets, email.

**Best for:** After finishing Template A, if you want to practice scheduled triggers instead of webhooks.

---

### Template C: Support Ticket Router (Variation)

**What it does:** Receives email/form via webhook → Groq categorizes (billing/tech/account) → logs to Google Sheets → emails the right team member based on category.

**Skills used:** Webhooks (Module 7), Groq classification (Module 3), Google Sheets, conditional routing.

**Best for:** After finishing Template A, if you want to practice AI classification and routing logic.

## Everyone: Start with Template A

Template A is the **primary capstone** with a full step-by-step build. Everyone should start there. Once it's working, Templates B and C are short variations that reuse the same pattern.

## What "Done" Looks Like

**Template A — Done means:**
- Make.com scenario is live and scheduled ON
- Google Sheet has at least 3 rows from test submissions
- Your inbox has 3+ emails with AI-generated titles and summaries
- You can explain each of the 4 steps in plain English

**Template B (optional) — Done means:**
- Scenario uses a scheduled trigger (instead of webhook)
- Runs at least 2 successful cycles
- Same Sheets + Email output as Template A

**Template C (optional) — Done means:**
- Groq successfully categorizes at least 3 test tickets
- Emails are routed to different recipients based on category
- Google Sheet shows category values correctly filled in

## Timeline Suggestion

Plan to spend **30–60 minutes** on Template A:

- **5 minutes** — Read through the whole Template A lesson before starting
- **10 minutes** — Set up the webhook and send test data
- **15 minutes** — Add Groq AI + Google Sheets + Email modules
- **10 minutes** — Run end-to-end, fix any errors, turn scenario ON
- **Optional: 15–30 minutes each** — Build Template B and C as variations

You do not need to finish in one sitting. It is totally fine to take a break and come back. Your automation tool saves your progress.

Do not worry about picking the "right" one. They all teach you how to connect triggers, AI, actions, and notifications into a working automation. The only difference is the scenario.

Ready? Pick your template and let's build something real!
