# Template C: Support Ticket Router

In this project, you will build an automation that handles incoming support emails:

1. **Receives** a new email via webhook
2. **Categorizes** the ticket using Claude AI
3. **Updates** a WordPress tracking page
4. **Notifies** the right team member

By the end, you will have a system that automatically sorts, tracks, and routes support requests.

## The Big Picture

```
[Webhook: email] → [Groq AI: categorize] → [Google Sheets: log] → [Email: route to owner]
```

This is a **variation of Template A** — same webhook-based pattern, but the AI step categorizes instead of summarizing, and the final email is routed based on the category. If you finished Template A, build this in 15 minutes by duplicating and swapping the prompts.

> **Recommended:** Build Template A first. The pattern is the same.

## Step 1: Set Up the Email Webhook

When a customer sends a support email, your email service can forward it to a webhook. This is the same concept from Module 7 — an event triggers your workflow.

1. In your automation tool, create a new workflow
2. Add a **Webhook** trigger
3. Configure your email service to forward support emails to the webhook URL

The webhook receives data like this:

```json
{
  "from": "customer@example.com",
  "subject": "Can't log in to my account",
  "body": "Hi, I've been trying to log in for the past hour but I keep getting an error message that says 'invalid credentials.' I've reset my password twice and it still doesn't work. This is urgent because I have a deadline today.",
  "timestamp": "2026-04-10T14:23:00Z"
}
```

**Expected result:** When you send a test email or trigger the webhook manually, you should see all four fields (from, subject, body, timestamp) appear in your automation tool's data panel.

**Troubleshooting:**
- **No data appears?** Make sure the workflow is in "listening" mode. In n8n, click "Listen for test event" before sending the test.
- **Only some fields show up?** Different email services send different fields. Check your email service's webhook documentation for the exact field names.
- **Getting duplicate triggers?** Some email services retry if they don't get a fast enough response. Add a deduplication step or increase your webhook's response timeout.

**Why this step matters:** Without automation, someone has to read every email and manually decide where to send it. The webhook starts the process instantly.

## Step 2: Categorize with Groq AI

Now AI reads the email and figures out what kind of issue it is and how urgent it is. This uses your prompting skills from Module 3 and the Groq setup from Module 7.

1. Add an **HTTP — Make a request** step (same pattern as Template A / Module 8)
2. Point it at the Groq API with your API key
3. Use this classification prompt inside the JSON body:

```
You are a support ticket classifier. Read this email and respond with JSON.

From: {{from}}
Subject: {{subject}}
Body: {{body}}

Classify it as:
- category: one of "billing", "technical", "account", "feature-request", "other"
- urgency: one of "low", "medium", "high"
- summary: a one-sentence summary of the issue
- assigned_to: suggest "billing-team", "dev-team", or "support-team"

Respond ONLY with valid JSON.
```

The AI responds with something like:

```json
{
  "category": "account",
  "urgency": "high",
  "summary": "Customer cannot log in despite multiple password resets",
  "assigned_to": "support-team"
}
```

**Expected result:** Groq returns a clean JSON object with all four fields filled in. Category and urgency should match the email content.

**Troubleshooting:**
- **AI returns text instead of JSON?** Add `"response_format": {"type": "json_object"}` to the request body (same trick as Template A).
- **Category is wrong?** Add examples in your prompt: "If the customer mentions payments or charges, category is 'billing'."
- **Urgency seems random?** Define criteria in the prompt: "high = customer is blocked, medium = frustrated but can work around it, low = general question."

**Why this step matters:** AI can classify hundreds of emails in seconds. A human doing the same work would take hours and might miscategorize some.

## Step 3: Log the Ticket to Google Sheets

Create a tracker spreadsheet first:

1. Go to [sheets.google.com](https://sheets.google.com) → **Blank spreadsheet**
2. Name it `Support Tickets`
3. Add headers: `Date`, `From`, `Subject`, `Category`, `Urgency`, `Summary`, `Assigned To`, `Status`

Then in Make:

1. Click `+` after the Groq step → search **Google Sheets** → **Add a Row**
2. Pick your `Support Tickets` sheet
3. Use the variable picker to map each column (same pattern as Template A):
   - **Date:** Now
   - **From:** Webhook → from
   - **Subject:** Webhook → subject
   - **Category / Urgency / Summary / Assigned To:** HTTP (Groq) response fields
   - **Status:** type `new`
4. Click **OK**

**Expected result:** Each test email appears as a new row in your sheet with the AI-filled category and urgency columns.

**Troubleshooting:**
- **Category/Urgency columns empty?** Groq response wasn't parsed. Toggle **Parse response: Yes** on the HTTP step.
- **Permission error:** Reconnect Google Sheets in Make and accept all permissions.

## Step 4: Route the Email Based on Category

Instead of Slack (which needs a paid setup), we'll use Make's **built-in Router** to send emails to different addresses depending on the category.

1. Click `+` after the Google Sheets step → pick **Flow Control → Router**
2. The Router splits your flow into multiple branches
3. On each branch, add an **Email → Send an Email** module
4. On each branch's **Filter** (the little wrench icon on the connection line), set a condition:

### Branch 1 — Billing
- **Filter condition:** HTTP (Groq) response → `category` **Equal to** `billing`
- **Email To:** your email (or `billing@yourcompany.com` if you own that address)
- **Subject:** `[BILLING] New ticket from [PICK: Webhook → from]`

### Branch 2 — Technical
- **Filter condition:** `category` **Equal to** `technical`
- **Email To:** your email
- **Subject:** `[TECH] New ticket from [PICK: Webhook → from]`

### Branch 3 — Account / Other
- **Filter condition:** `category` **Equal to** `account` (or no filter = fallback)
- **Email To:** your email
- **Subject:** `[SUPPORT] New ticket from [PICK: Webhook → from]`

Use the same HTML email body on all three branches (variable picker for category, urgency, summary, from).

**Expected result:** When you send a billing-style test email, only the billing branch fires. Send a tech-style email, only the tech branch fires. Your Google Sheet tracks all of them.

**Troubleshooting:**
- **Multiple branches fire for one email:** Filter conditions overlap. Make each condition mutually exclusive.
- **No branch fires:** Category value coming from Groq doesn't match the filter exactly. Use Make's execution log to see the real value — might be `"Billing"` (capitalized) instead of `"billing"`. Adjust the filter or add `toLowerCase()` to the prompt.
- **No email received:** The fallback branch needs an "else" filter (leave it empty to catch everything unrouted).

**Why this step matters:** Routing is the core of real automation. The same pattern — AI decision → conditional branching → different actions — powers lead routing, moderation queues, fraud detection, and more.

## Putting It All Together

Your complete workflow handles support emails end to end:

1. **Email arrives** &rarr; webhook fires
2. **Groq AI** &rarr; reads, categorizes, and assigns
3. **Google Sheets** &rarr; logs the ticket for tracking
4. **Router + Email** &rarr; alerts the right team member based on category

What used to require a person reading every email, deciding the category, updating a spreadsheet, and forwarding to the right team — now happens automatically in seconds.
