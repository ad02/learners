# Template C: Support Ticket Router

In this project, you will build an automation that handles incoming support emails:

1. **Receives** a new email via webhook
2. **Categorizes** the ticket using Claude AI
3. **Updates** a WordPress tracking page
4. **Notifies** the right team member

By the end, you will have a system that automatically sorts, tracks, and routes support requests.

## The Big Picture

```
[Webhook: email] → [Groq AI: categorize] → [Google Sheets: log] → [Email: route to owner]
```

This is a **variation of Template A** — same webhook-based pattern, but the AI step categorizes instead of summarizing, and the final email is routed based on the category. If you finished Template A, build this in 15 minutes by duplicating and swapping the prompts.

> **Recommended:** Build Template A first. The pattern is the same.

## Step 1: Set Up the Email Webhook

When a customer sends a support email, your email service can forward it to a webhook. This is the same concept from Module 7 — an event triggers your workflow.

1. In your automation tool, create a new workflow
2. Add a **Webhook** trigger
3. Configure your email service to forward support emails to the webhook URL

The webhook receives data like this:

```json
{
  "from": "customer@example.com",
  "subject": "Can't log in to my account",
  "body": "Hi, I've been trying to log in for the past hour but I keep getting an error message that says 'invalid credentials.' I've reset my password twice and it still doesn't work. This is urgent because I have a deadline today.",
  "timestamp": "2026-04-10T14:23:00Z"
}
```

**Expected result:** When you send a test email or trigger the webhook manually, you should see all four fields (from, subject, body, timestamp) appear in your automation tool's data panel.

**Troubleshooting:**
- **No data appears?** Make sure the workflow is in "listening" mode. In n8n, click "Listen for test event" before sending the test.
- **Only some fields show up?** Different email services send different fields. Check your email service's webhook documentation for the exact field names.
- **Getting duplicate triggers?** Some email services retry if they don't get a fast enough response. Add a deduplication step or increase your webhook's response timeout.

**Why this step matters:** Without automation, someone has to read every email and manually decide where to send it. The webhook starts the process instantly.

## Step 2: Categorize with Claude AI

Now AI reads the email and figures out what kind of issue it is and how urgent it is. This uses your prompting skills from Module 3 and 4.

1. Add a **Claude AI** step
2. Write a classification prompt:

```
You are a support ticket classifier. Read this email and respond with JSON.

From: {{from}}
Subject: {{subject}}
Body: {{body}}

Classify it as:
- category: one of "billing", "technical", "account", "feature-request", "other"
- urgency: one of "low", "medium", "high"
- summary: a one-sentence summary of the issue
- assigned_to: suggest "billing-team", "dev-team", or "support-team"

Respond ONLY with valid JSON.
```

The AI responds with something like:

```json
{
  "category": "account",
  "urgency": "high",
  "summary": "Customer cannot log in despite multiple password resets",
  "assigned_to": "support-team"
}
```

**Expected result:** Claude returns a clean JSON object with all four fields filled in correctly. The category and urgency should match the content of the email.

**Troubleshooting:**
- **AI returns text instead of JSON?** Strengthen your prompt: add "You must respond with ONLY a JSON object. No other text, no explanations, no markdown formatting."
- **Category is wrong?** Provide examples in your prompt. Add: "Example: if the customer mentions payments or charges, the category is 'billing'."
- **Urgency seems random?** Define your urgency criteria: "high = customer is blocked or mentions a deadline, medium = customer is frustrated but can work around it, low = general question or feedback."

**Why this step matters:** AI can read and categorize hundreds of emails in minutes. A human doing the same work would take hours and might miscategorize some.

## Step 3: Update the WordPress Tracker

Now we log the ticket on a WordPress page so the team has a central tracking dashboard. This uses the WordPress REST API from Module 6 and 7.

1. Add a **WordPress** step
2. Update your support tracker page with the new ticket:

```
POST https://yoursite.com/wp-json/wp/v2/posts
{
  "title": "Ticket: Can't log in to my account",
  "content": "<table>
    <tr><td>From</td><td>customer@example.com</td></tr>
    <tr><td>Category</td><td>Account</td></tr>
    <tr><td>Urgency</td><td>High</td></tr>
    <tr><td>Assigned</td><td>Support Team</td></tr>
    <tr><td>Summary</td><td>Customer cannot log in despite multiple password resets</td></tr>
  </table>",
  "status": "publish",
  "categories": [support-tracker-category-id]
}
```

**Expected result:** A new post appears in your WordPress admin under the "Support Tracker" category. It should display a clean table with all the ticket details. When viewed on the front end, the table should be readable and organized.

**Troubleshooting:**
- **Table looks broken?** Make sure the HTML is valid. Check for missing closing tags (`</td>`, `</tr>`, `</table>`).
- **Post not appearing?** Check the category ID. Also verify that your WordPress user has permission to publish posts.
- **Content shows HTML tags instead of a formatted table?** Your WordPress theme might be stripping HTML. Try using the "Classic" editor block or add the content inside a `<!-- wp:html -->` block.

> **Pro tip:** Create a "Support Tracker" category in WordPress before building this step. Set it to not appear on your main blog feed if you do not want tickets visible to the public.

**Why this step matters:** WordPress becomes your ticket database. Anyone on the team can check the tracker page to see all open tickets, search by category, and track patterns.

## Step 4: Notify the Right Person

Finally, send a notification to the team member who should handle this ticket (Module 8 notification pattern).

1. Add a **Notification** step
2. Route based on the AI's `assigned_to` value:

### Routing Logic Examples

Here is how the conditional routing works in detail:

```
IF assigned_to == "billing-team":
  Send email to billing@yourcompany.com
  Send Slack message to #billing channel
  Include: invoice lookup link, customer email

ELSE IF assigned_to == "dev-team":
  Send email to dev@yourcompany.com
  Send Slack message to #engineering channel
  Include: error details, steps to reproduce

ELSE IF assigned_to == "support-team":
  Send email to support@yourcompany.com
  Send Slack message to #support channel
  Include: customer name, urgency level, summary
```

In your automation tool, this looks like an **IF node** or **Router** that checks the `assigned_to` value and sends the notification to the correct channel.

The notification message:

```
New support ticket assigned to you!
Category: Account (High urgency)
Summary: Customer cannot log in despite multiple password resets
From: customer@example.com
Tracker: {{wordpress_post_link}}
```

**Expected result:** The correct team channel receives the notification. If the ticket is categorized as "billing", the billing channel gets the message — not the dev or support channels.

**Troubleshooting:**
- **Notification goes to the wrong channel?** Check the IF condition. Make sure you are comparing the exact string value (e.g., "billing-team" not "Billing Team").
- **All tickets go to the same channel?** Your IF conditions might not be working. Test with different email content to confirm the AI is producing different `assigned_to` values.
- **No notification at all?** Check that your Slack bot has permission to post in the target channel. Try posting a test message to the channel manually first.

**Why this step matters:** The right person gets notified immediately. No more emails sitting in a shared inbox waiting for someone to claim them.

## Putting It All Together

Your complete workflow handles support emails end to end:

1. **Email arrives** &rarr; webhook fires
2. **Claude AI** &rarr; reads, categorizes, and assigns
3. **WordPress** &rarr; logs the ticket for tracking
4. **Notification** &rarr; alerts the right team member

What used to require a person reading every email, deciding the category, updating a spreadsheet, and forwarding to the right team — now happens automatically in seconds.
