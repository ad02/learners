# Template A: Blog Auto-Publisher

In this project, you will build an automation that does four things in sequence:

1. **Receives** a form submission via webhook
2. **Summarizes** the content using Claude AI
3. **Publishes** a draft post to WordPress
4. **Notifies** your team on Slack

By the end, you will have a complete workflow that turns raw form submissions into polished blog drafts — automatically.

## Step 1: Set Up the Webhook Trigger

Remember webhooks from Module 7? A webhook is like a doorbell — when someone submits the form, it "rings" your automation.

Here is what you need to do:

1. In your automation tool (like n8n, Make, or Zapier), create a new workflow
2. Add a **Webhook** trigger as the first step
3. Set it to accept POST requests
4. Copy the webhook URL — you will paste this into your form

The webhook will receive data like this:

```json
{
  "name": "Jane Smith",
  "topic": "How AI is changing small business",
  "content": "AI tools are making it easier for small businesses to compete with larger companies. From automated customer service to content creation..."
}
```

**Expected result:** After setting up the webhook and sending a test request, you should see the JSON data appear in your automation tool's execution panel. Each field (name, topic, content) should be visible and selectable for use in later steps.

**Troubleshooting:**
- **No data appears?** Make sure the webhook URL is correct and the workflow is active/listening. In n8n, you need to click "Listen for test event" before sending the test request.
- **Data shows as empty?** Check that your test tool is sending the `Content-Type: application/json` header.
- **Getting a 404 error?** The webhook URL may have expired. Create a new one and try again.

**Why this step matters:** Without a trigger, nothing starts. The webhook is the "when" — when a form is submitted, do everything else.

## Step 2: Connect Claude AI to Summarize

Now we connect Claude AI to process the form submission. This is where your prompting skills from Module 3 and 4 come in.

1. Add an **AI** step after the webhook
2. Connect it to Claude (via API key)
3. Write a prompt like this:

```
You are a blog editor. Take the following submission and create a polished blog post summary.

Topic: {{topic}}
Raw content: {{content}}

Write a 2-3 paragraph summary that is engaging and professional. Include a suggested title.
```

**Expected result:** Claude returns a polished summary. For the test data above, you might see:

```
Title: "How AI Levels the Playing Field for Small Businesses"

Small businesses have long struggled to compete with larger corporations that have bigger budgets and more staff. But artificial intelligence is changing that equation. Today, even a one-person shop can use AI tools to automate customer service, create professional content, and streamline operations — tasks that used to require entire departments.

From chatbots that handle customer questions 24/7 to AI writing assistants that draft marketing emails in seconds, these tools are giving small businesses superpowers they never had before...
```

**Troubleshooting:**
- **AI returns an error?** Check your API key. Make sure it is valid and has not expired.
- **Response is too long or too short?** Adjust your prompt. Add "Keep it under 200 words" or "Write at least 3 paragraphs" to guide the length.
- **Response does not include a title?** Make sure your prompt explicitly asks for one. AI follows instructions literally.

**Why this step matters:** Raw form submissions are messy. AI turns them into something publishable without anyone having to manually edit.

## Step 3: Create a WordPress Draft Post

Now we use the WordPress REST API (Module 6 & 7) to create a draft post.

1. Add a **WordPress** action step
2. Connect it to your WordPress site using the REST API
3. Set it to create a new post with:
   - **Title:** The AI-generated title
   - **Content:** The AI-generated summary
   - **Status:** "draft" (so a human can review before publishing)
   - **Author:** Set to your default author

The API call looks like this:

```
POST https://yoursite.com/wp-json/wp/v2/posts
{
  "title": "AI-Generated Title Here",
  "content": "The polished summary from Claude...",
  "status": "draft"
}
```

**Expected result:** WordPress responds with the created post data, including:

```json
{
  "id": 789,
  "title": { "rendered": "How AI Levels the Playing Field for Small Businesses" },
  "status": "draft",
  "link": "https://yoursite.com/?p=789"
}
```

You should be able to log into your WordPress admin, go to **Posts > All Posts**, and see the new draft.

**Troubleshooting:**
- **401 Unauthorized?** Your WordPress credentials are wrong. Double-check your username and application password. (Remember: you need an Application Password, not your regular login password.)
- **404 Not Found?** The site URL is wrong, or the REST API is disabled. Check if `yoursite.com/wp-json/wp/v2/posts` works in your browser.
- **Post appears but content is empty?** Make sure you are mapping the AI output to the content field correctly. Check variable names in your automation tool.

**Why this step matters:** Automating the draft creation saves time. The post is ready for review — no copy-pasting required.

## Step 4: Send a Slack Notification

Finally, let the team know a new draft is ready (Module 8 notification pattern).

1. Add a **Slack** notification step
2. Connect it to your Slack workspace
3. Send a message to your content channel:

```
New blog draft ready for review!
Title: {{post_title}}
Author: {{submitter_name}}
Link: {{wordpress_edit_link}}
```

**Expected result:** A message appears in your Slack channel within seconds. It should include the post title, the submitter's name, and a link to edit the draft in WordPress.

**Troubleshooting:**
- **Message not appearing?** Check that the bot is added to the channel. In Slack, type `/invite @yourbotname` in the channel.
- **Message appears but link is wrong?** Make sure you are using the WordPress edit link (usually `yoursite.com/wp-admin/post.php?post=ID&action=edit`), not the public post link.

**Why this step matters:** Without notification, the draft sits there unnoticed. The Slack message closes the loop — someone knows to review it.

## Putting It All Together

Your complete workflow now looks like this:

1. **Form submitted** &rarr; webhook fires
2. **Claude AI** &rarr; summarizes and polishes the content
3. **WordPress API** &rarr; creates a draft post
4. **Slack** &rarr; notifies the team

That is four tools working together automatically. What used to take 15-20 minutes of manual work now happens in seconds.
