# Template A: Content Pipeline

This is the **primary capstone project**. You'll build a real, running automation on Make.com's free tier that:

1. **Receives** a submission via webhook
2. **Summarizes** it with Groq AI
3. **Logs** the result to a Google Sheet (your draft queue)
4. **Emails** you a notification

Every tool is free. Total build time: ~30 minutes.

## What You'll Need (All Free)

- ✅ Make.com account ([from Module 8](https://www.make.com))
- ✅ Groq API key ([from Module 7](https://console.groq.com))
- ✅ Google account (for Sheets — probably already have this)
- ✅ An email address

## The Workflow

```
[Webhook] → [Groq AI] → [Google Sheets: append row] → [Email: notify]
```

Four steps, no WordPress-specific setup headaches. The Google Sheet becomes your **review queue** — just like draft posts, but on the free tier.

---

## Step 1 — Create the Make Scenario

1. Log in to [Make.com](https://www.make.com) → **Scenarios** → **Create a new scenario**
2. Click the big `+` → search **Webhooks** → pick **Custom webhook**
3. **Add** → name it `content-pipeline` → **Save**
4. **Copy the webhook URL** somewhere safe

**Expected result:** Make says "The webhook is listening for data." Leave this tab open.

**Troubleshooting:**
- Can't find Webhooks module? Search lowercase: `webhook`
- Webhook URL changed? Make regenerates if you delete and recreate — always use the current one

---

## Step 2 — Send Test Data

In a new browser tab, open DevTools (F12) → **Console** tab → paste and run:

```js
fetch("PASTE_YOUR_WEBHOOK_URL", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({
    name: "Jane Smith",
    email: "jane@example.com",
    topic: "How AI helps small business",
    content: "AI tools like ChatGPT and Claude are changing how small businesses operate. What used to require a team of 5 can now be done by one person with the right tools. This is especially true for content creation, customer service, and data analysis."
  })
});
```

Go back to Make → the webhook bubble should now show the sample data. Click **OK**.

**Expected result:** Make displays your JSON data with each field individually mapped (name, email, topic, content).

**Troubleshooting:**
- Webhook didn't fire? Check the URL has no typos. Re-copy it.
- Getting CORS error in console? That's normal — Make still received the request. Check the Make tab.

---

## Step 3 — Add Groq AI

1. Click the `+` after the webhook → search **HTTP** → **Make a request**
2. Configure:
   - **URL:** `https://api.groq.com/openai/v1/chat/completions`
   - **Method:** `POST`
   - **Headers:** add 2 items
     - `Authorization` = `Bearer YOUR_GROQ_KEY` (your real key)
     - `Content-Type` = `application/json`
   - **Body type:** Raw | **Content type:** JSON (application/json)
   - **Request content:**

```json
{
  "model": "llama-3.1-8b-instant",
  "messages": [
    {"role": "system", "content": "You are a blog editor. Given a topic and raw content, write a polished 2-paragraph blog-style summary. Respond with JSON: {\"title\": \"...\", \"summary\": \"...\"}"},
    {"role": "user", "content": "Topic: {{1.topic}}\n\nContent: {{1.content}}"}
  ],
  "response_format": {"type": "json_object"}
}
```

> 💡 **Variable mapping:** After clicking into the Request content field, Make shows variable pills like `{{1.topic}}` — click them in the right panel to insert instead of typing them.

3. **Parse response:** scroll down → toggle **Parse response** to **Yes**
4. Click **OK**

**Expected result:** After a test run, Groq returns a JSON response with `title` and `summary` fields — accessible as variables in later steps.

**Troubleshooting:**
- **401 Unauthorized:** API key wrong or has extra spaces. Copy/paste fresh from Groq console.
- **No variables show up from Groq:** Parse response must be **Yes** — rerun the scenario after toggling.
- **Response cut off:** Add `"max_tokens": 500` to the body.

---

## Step 4 — Create a Google Sheet

In a new tab:

1. Go to [sheets.google.com](https://sheets.google.com) → **Blank spreadsheet**
2. Name it `Content Queue`
3. In row 1, add these headers (one per column): `Date`, `Name`, `Email`, `Topic`, `Title`, `Summary`, `Status`
4. Save — don't close

---

## Step 5 — Add Google Sheets Module

Back in Make:

1. Click `+` after the Groq step → search **Google Sheets** → pick **Add a Row**
2. Click **Create a connection** → sign in with your Google account → allow access
3. **Spreadsheet:** click the dropdown → find your `Content Queue` sheet
4. **Sheet Name:** Sheet1 (default)
5. **Column values** — map each column:
   - **Date:** click the variable picker → pick **Now** (under Date)
   - **Name:** `{{1.name}}`
   - **Email:** `{{1.email}}`
   - **Topic:** `{{1.topic}}`
   - **Title:** from Groq response → extract `.title` (the parsed JSON field)
   - **Summary:** from Groq response → extract `.summary`
   - **Status:** type `pending review`
6. Click **OK**

**Expected result:** After running, a new row appears in your Google Sheet with all fields filled.

**Troubleshooting:**
- **Permission denied on Sheets:** sign out of the Google connection in Make and reconnect, accepting all permissions.
- **Columns misaligned:** recount — the header row must match what you map. Case-sensitive isn't required but field order matters.
- **Title/Summary show as `[object Object]`:** Groq's JSON wasn't parsed. Go back to Step 3 and verify "Parse response" is on.

---

## Step 6 — Add Email Notification

1. Click `+` after Sheets → search **Email** → **Send an Email**
2. Create a connection (Make's built-in email — no SMTP config needed)
3. Configure using the **variable picker** on the right (don't type variable names by hand — click them):
   - **To:** your email address
   - **Subject:** `New content: ` then click the **HTTP (Groq)** module in the variable picker → pick **title**
   - **Content type:** HTML
   - **Content:** paste the template below, then replace each `[PICK: …]` with the actual variable from the picker panel:

```html
<h2>[PICK: HTTP module → title]</h2>
<p><strong>From:</strong> [PICK: Webhook → name] ([PICK: Webhook → email])</p>
<p><strong>Summary:</strong></p>
<p>[PICK: HTTP module → summary]</p>
<p><a href="https://docs.google.com/spreadsheets/">Review in your Content Queue</a></p>
```

> 💡 **Why not type `{{2.title}}` directly?** Make.com uses numbered module references (e.g., `{{2.title}}` = module 2's `title` field). Module numbers shift if you rearrange modules, so always use the variable picker — Make will insert the correct reference automatically.

4. Click **OK**

---

## Step 7 — Run End-to-End

1. Click **Run once** (bottom-left in Make)
2. Re-send the DevTools fetch from Step 2
3. Watch all 4 bubbles turn green
4. Check: Google Sheet has a new row ✅
5. Check: Your inbox has the summary email ✅

**If anything failed:** Click the red bubble → the error pane shows the exact issue. Fix, then Run Once again.

---

## Step 8 — Turn It On

Flip **Scheduling** (bottom-left) from **OFF** to **ON**.

Your automation now runs 24/7. Anyone who sends data to your webhook URL triggers the full pipeline. **You built a real content operations tool.**

## Extensions (Optional)

Want to go deeper? Try adding:

- **A filter:** Only email you if the topic contains certain keywords (Make has a Filter module between steps)
- **A second AI step:** Classify the submission as "blog / question / complaint" before storing
- **A Discord/webhook.site notification** instead of email for team channels
- **A cron schedule** that summarizes all "pending review" rows every morning
